# 📘 1. Resilience  
  
The ability of a microservice system to continue working even when some services fail.  
  
Resilience is the capability of microservices to handle failures and still provide acceptable service by using patterns like retries, circuit breakers, timeouts, and fallbacks.  
  
⚙️**** Common Resilience Techniques****  
———————————————————————  
**1️⃣ Circuit Breaker**  
Stops calling a failing service.  
States:  
* Closed → normal  
* Open → block calls  
* Half-open → test recovery  
  
**2️⃣ Retry**  
Retry failed requests (with limits + backoff).  
  
**3️⃣ Timeout**  
Don’t wait forever for a response.  
  
**4️⃣ Fallback**  
Return default data when service fails.  
Example:  
```
Order Service down → show cached orders

```
  
**5️⃣ Bulkhead**  
Isolate resources so one service can’t exhaust everything. Each service gets its own thread pool:  
Payment Pool  
Order Pool  
User Pool  
So:  
Payment failure ❌ User service still works ✅  
  
  
6️⃣** Rate Limiting**  
Controlling how many requests a client (user / IP / service) is allowed to make in a given time window.
