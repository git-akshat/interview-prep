# 📘 5. Service Discovery  
  
> **Service Discovery is a mechanism that allows microservices to automatically find each other’s network locations (IP/port) instead of hardcoding them.**  
> 
> In microservices:  

* Services scale up/down  
* IPs change  
* Containers restart  
> So this **will NOT work** reliably:  
>   
```
http://localhost:8081

```
> Service Discovery solves this.  
>   
> **🧠 Why do we need it?**  
> 
> Because in production:  
>   
> Payment Service  
> 
> → multiple instances  
> 
> → dynamic IPs  
> 
> → auto scaling  
> 
> Hardcoding addresses causes failures.  
> 
> Service Discovery provides:  
> 
> 👉 Dynamic lookup 👉 Load balancing 👉 Fault tolerance  
>   
> **🔁 How Service Discovery Works (High Level)**  
> 
> **1️⃣ Service registers itself**  
> 
> When a service starts:  
```
Payment Service → registers with Registry

```
>   
> **2️⃣ Client asks registry**  
> 
> Caller asks:  
```
Where is Payment Service?

```
> Registry replies with available instances.  
>   
> **3️⃣ Client calls service**  
```
Client picks one instance (often load balanced).

```
>   
> 🧩**** Core Components****  
>   
> **🔹 Service Registry**  
> 
> Central directory of services.  
>   
> Example tools:  

* Netflix Eureka  
* Consul  
* Apache Zookeeper  
>   
> **🔹 Service Provider**  
> 
> Registers itself.  
>   
> **🔹 Service Consumer**  
> 
> Queries registry and calls provider.  
  
  
> 🧭**** Two Types of Service Discovery****  
>   
> ✅** Client-Side Discovery**  
> 
> Client decides which service instance to call.  
> 
> Flow:  
```
Client → Registry → Service Instance

```
> Used by:  

* Netflix Eureka  
> Spring Cloud + Eureka + Feign = classic combo.  
>   
> ✅** Server-Side Discovery**  
> 
> Client hits a router/load balancer.  
> 
> Flow:  
```
Client → Load Balancer → Service

```
> Examples:  

* Kubernetes  
* AWS Elastic Load Balancing  
> Client doesn’t know registry.  
>   
> ⚙️**** Spring Boot + Eureka Example (Conceptual)****  
  
**Service registers:**  
```
eureka:
  client:
    service-url:
      defaultZone: http://localhost:8761/eureka
 
spring:
  application:
    name: payment-service

```
>   
```
http://payment-service/payment

```
> No IP needed.
